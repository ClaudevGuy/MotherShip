§ The Subject Three briefs on what Mothership is, and isn't. p. 01
I.

It is a control plane, not another framework.

You already chose your framework — CrewAI, LangGraph, Paperclip, OpenAI's Agent SDK, or the 400 lines of homemade TypeScript that refuses to die. Mothership sits above all of them. One console, every run, every cost.

II.

It is self-hosted, which is the point.

Your prompts are product. Your logs are leverage. Both belong on your infrastructure, not in a third-party database priced per seat. Mothership runs on Postgres and Vercel, or any Node host, or Docker. Your keys stay yours.

III.

It is opinionated about cost.

Every request is profiled and routed to the cheapest tier that will pass. Opus for the work that matters. Haiku for the classification most teams overpay for. The savings show up on each run, and summarize for the month.

Long form · The Intelligence

Every request, routed to the cheapest model that works.

The routing engine profiles each task on the way in — task complexity, expected output size, agent category, whether the run touches production — and selects the lowest tier that will still pass. Production-critical paths get Tier 1. The classifications, summaries, and reformatting that most teams quietly overpay for get Tier 3.

The engine's reasoning is recorded on every call. If it upgrades a run mid-month, you see why. If it downgrades from Opus to Haiku on a reformatting job, you see the reasoning and the eval score that permitted it.

The savings are not a marketing number. They appear on every run as "Saved $X.XX vs Tier 1," and they summarize on the dashboard per agent, per team, per month. Finance stops asking. Engineering stops defending.

— The routing thesis, in one line

The routing engine ships with a default policy tuned for developer workloads — classification, summarization, code review, documentation, triage. It can be overridden per-agent, per-request, or disabled entirely. Source code in /lib/model-selector.ts, opinions welcome.

§ 02

Bring what you already built.

Mothership ships a lightweight SDK that ingests run events from any framework. Your CrewAI pipeline, your LangGraph graph, your Paperclip workflow, your bespoke Python — fourteen lines and each of them shows up on the console. Full cost tracking, run history, eval coverage. No rewrites.

Field notes from the operations desk.

A partial inventory. Read the docs for the exhaustive one, or clone the repository and read the code — whichever you trust more.

No. 01 — Evaluation

Two scorers, no theatre.

Deterministic string matches for facts — "mentions pricing," "under 150 words." Claude-as-judge for tone. Scores sparkline per case, per suite, per run. You see regressions before the customer does.

42 cases · 94% passing
No. 02 — Workflows

Pipelines as flight plans.

Drag triggers, chain agents, pipe the output of one into the input of the next. Manual, cron, webhook, event-driven. Every run records a step-level replay for when something goes sideways.

14 workflows · 2,847 runs
No. 03 — Prompt Studio

Versioning that behaves like git.

Save, diff, activate. Each version is immutable. Activating a version changes what the agent uses on the very next run — no deploy. Roll back with one click. Link a prompt to an agent and upgrade it in flight.

v12 active · 42 versions
No. 04 — Audit

Everything, forever.

Every create, update, delete — who, when, what. Searchable. Exportable. Signed. The answer to "who changed the prompt last Friday" takes six seconds, not forty minutes.

Immutable · scoped · queryable
No. 05 — Cost forecast

A quiet crystal ball.

Trailing-7-day average, week-over-week trend arrow. Projected monthly spend against budget with a red line you cannot miss. When routing catches a 3× upgrade, it's flagged for review.

$389 projected · under budget
No. 06 — Kill switch

One button, no ceremony.

For the moment a prompt change drops a zero off a pricing field, or an agent decides to recurse forever at 3 a.m. Admin-scoped, confirmed, audit-logged. Everything stops in under a second.

Admin · confirmed · logged
The editor's note

Self-host it.
Own your data.
Pay nothing.

Mothership was built in a worktree, over two weeks, by one person who was tired of stitching Langfuse, Vanta, Retool, and a Google Sheet together. It ships with a schema, an SDK, a CLI, and a set of opinions that will either match yours or won't.

There is no signup. There is no free tier that turns into a paywall at the worst moment. There is a repository, a docker-compose, and a README that fits on one screen. Clone it, read it, keep it.

If you extend it, send a pull request. If you disagree with a routing heuristic, fork it. This is a publication you edit as you read.
